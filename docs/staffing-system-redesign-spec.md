# スタッフ・人員配置管理システム 再設計仕様書

**作成日**: 2026年2月11日
**バージョン**: 1.0（ドラフト）
**ステータス**: 要件定義フェーズ

---

## 1. 背景と目的

### 1.1 現状の問題

| カテゴリ | 問題 | 影響 |
|---------|------|------|
| 機能重複 | 人員配置設定が4箇所に分散 | どこで設定すればいいか不明 |
| 巨大ファイル | StaffView(3,299行), StaffManagementView(4,996行) | 保守困難、バグ発生リスク |
| メニュー混乱 | 6つの類似メニューが並列 | ユーザーが迷う |
| 概念の曖昧さ | 「権限」「役職」「人員区分」が混在 | 設定の整合性が取れない |

### 1.2 再設計の目的

1. **シンプル化**: 6メニュー → 3メニューへ統合
2. **明確化**: 各画面の責務を明確に分離
3. **一貫性**: 設定箇所の一本化（Single Source of Truth）
4. **保守性**: 1ファイル1,000行以下を目標

---

## 2. 用語定義

| 用語 | 定義 | 備考 |
|------|------|------|
| **スタッフ** | 施設で働く職員 | ユーザーアカウントと紐づく |
| **人員区分** | 基準人員 or 加算人員 | 報酬計算に影響 |
| **勤務形態** | 常勤専従 / 常勤兼務 / 非常勤 | 配置基準の判定に使用 |
| **管理者** | 施設の管理責任者 | 1名、他職種と兼務可 |
| **児発管** | 児童発達支援管理責任者 | 1名、常勤専従必須 |
| **FTE** | 常勤換算（週労働時間÷所定労働時間） | 例: 30h÷40h = 0.75 |
| **シフト** | 日ごとの出勤予定 | カレンダー形式で管理 |
| **勤務体制一覧表** | 行政提出用の人員配置表 | 月次で作成 |

## 4. 画面詳細仕様

### 4.1 スタッフマスタ

#### 4.1.1 概要

| 項目 | 内容 |
|------|------|
| 目的 | スタッフの基本情報を一元管理 |
| 対象ユーザー | 施設管理者 |
| URL | /business → staff-master |

#### 4.1.2 機能一覧

| 機能 | 説明 | 優先度 |
|------|------|--------|
| スタッフ一覧 | 登録済みスタッフの一覧表示・検索・フィルタ | 必須 |
| スタッフ登録 | 新規スタッフの追加（招待メール送信） | 必須 |
| スタッフ編集 | 基本情報・資格・経験年数の編集 | 必須 |
| スタッフ削除 | スタッフの削除（論理削除） | 必須 |
| 研修記録 | スタッフごとの研修履歴管理 | 必須 |
| 履歴書出力 | 実務経験証明書のPDF出力 | 任意 |
| 有給管理 | 有給残日数・取得履歴 | 任意 |

#### 4.1.3 画面レイアウト

```
┌─────────────────────────────────────────────────────────────┐
│ スタッフマスタ                           [+ スタッフ追加]   │
├─────────────────────────────────────────────────────────────┤

├─────────────────────────────────────────────────────────────┤
│ │ 名前       │ 職種       │ 資格         │ 経験  │ 操作 │ │
│ ├────────────┼────────────┼──────────────┼───────┼──────┤ │

```

#### 4.1.4 データ項目

| 項目 | 型 | 必須 | 説明 |
|------|----|------|------|
| 氏名（姓） | string | ○ | |
| 氏名（名） | string | ○ | |
| メールアドレス | string | ○ | ログイン用 |
| 電話番号 | string | | |
| 職種 | enum | ○ | 児童指導員/保育士/PT/OT/ST/その他 |
| 資格 | string[] | | 複数選択可 |
| 経験年数 | number | | 障害福祉サービス経験 |
| 入職日 | date | | |
| 備考 | text | | |

#### 4.1.5 除外する機能（人員配置へ移動）

以下は「人員配置」画面で管理するため、スタッフマスタからは**除外**:

- 人員区分（基準人員/加算人員）
- 勤務形態（常勤専従/常勤兼務/非常勤）
- 週所定労働時間
- 管理者フラグ
- 児発管フラグ

## 6. 制度要件（児発・放デイ）

### 6.1 人員配置基準

| 職種 | 配置基準 | 備考 |
|------|----------|------|
| 管理者 | 1名以上 | 非常勤可、他職種と兼務可 |
| 児発管 | 1名以上 | **常勤専従必須**、管理者とのみ兼務可 |
| 児童指導員・保育士 | 2名以上 | うち1名は常勤、定員10名以下の場合 |

### 6.2 サービス提供時間中の配置

- **2名以上**配置必須
- うち**1名は常勤専従**

### 6.3 常勤換算の計算

```
FTE = 個人の週労働時間 ÷ 施設の週所定労働時間（通常40時間）
```
